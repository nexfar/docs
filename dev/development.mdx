<Info>
  **Pré-requisitos**:
  - Node.js versão 19 ou superior
  - Um repositório de documentos com um arquivo `docs.json`
</Info>

Siga estes passos para instalar e executar o Mintlify no seu sistema operacional.

<Steps>
<Step title="Instale o Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Visualize localmente">

Navegue até o diretório dos seus documentos onde o arquivo `docs.json` está localizado e execute o seguinte comando:

```bash
mint dev
```

Uma visualização local da sua documentação estará disponível em `http://localhost:3000`.

</Step>
</Steps>

## Portas personalizadas

Por padrão, o Mintlify usa a porta 3000. Você pode personalizar a porta em que o Mintlify é executado usando a flag `--port`. Por exemplo, para executar o Mintlify na porta 3333, use este comando:

```bash
mint dev --port 3333
```

Se você tentar executar o Mintlify em uma porta que já está em uso, ele usará a próxima porta disponível:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versões do Mintlify

Observe que cada versão do CLI está associada a uma versão específica do Mintlify. Se sua visualização local não estiver alinhada com a versão de produção, atualize o CLI:

```bash
npm mint update
```

## Validando links

O CLI pode auxiliar na validação de links na sua documentação. Para identificar links quebrados, use o seguinte comando:

```bash
mint broken-links
```

## Formatação de código

Sugerimos usar extensões no seu IDE para reconhecer e formatar MDX. Se você é usuário do VSCode, considere a [extensão MDX para VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para realce de sintaxe e [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para formatação de código.

## Solução de problemas

<AccordionGroup>
  <Accordion title='Erro: Could not load the "sharp" module using the darwin-arm64 runtime'>

    Isso pode ser devido a uma versão desatualizada do node. Tente o seguinte:
    1. Remova a versão atualmente instalada do CLI: `npm remove -g mint`
    2. Atualize para Node v19 ou superior.
    3. Reinstale o CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Problema: Encontrando um erro desconhecido">

    Solução: Vá para a raiz do seu dispositivo e exclua a pasta `~/.mintlify`. Em seguida, execute `mint dev` novamente.
  </Accordion>
</AccordionGroup>

Curioso sobre o que mudou na versão mais recente do CLI? Confira o [changelog do CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).

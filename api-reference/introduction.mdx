# API de Ingestão Nexfar

Documentação completa e interativa dos endpoints REST para envio de dados à plataforma Nexfar.

## Autenticação

Todos os endpoints requerem autenticação JWT via header:

```
Authorization: Bearer {seu-token-jwt}
```

**Formato do Token:**
- Type: Bearer JWT
- Claims obrigatórios: `client_id`, `environment`, `scopes`
- Validade: 24 horas

## Características da API

<AccordionGroup>
  <Accordion title="Rate Limiting">
    - **Limite:** 100 requisições por minuto por cliente
    - **Headers:** `X-RateLimit-Remaining`, `X-RateLimit-Reset`
    - **Status 429:** Quando limite excedido
  </Accordion>

  <Accordion title="Rastreabilidade">
    - Cada requisição gera `correlation_id` único
    - Histórico completo em audit log
    - Hash SHA-256 do payload para integridade
  </Accordion>

  <Accordion title="Respostas Padronizadas">
    - **201 Created:** Sucesso na criação
    - **400 Bad Request:** Dados inválidos (com detalhes)
    - **401 Unauthorized:** JWT inválido ou ausente
    - **429 Too Many Requests:** Rate limit excedido
  </Accordion>
</AccordionGroup>
